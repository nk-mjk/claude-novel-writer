Chapter 02

Cowork는 이미 하네스다

1장에서 Claude Code의 하네스를 5개 레이어로 살펴봤습니다. 그런데 Cowork도 동일한 구조 위에 있습니다. 차이는 하나입니다 — 대부분의 레이어가 이미 내장되어 있어서 따로 설정할 필요가 없다는 것입니다.

레이어 Cowork 상태 설명
Layer 1 Tools 내장 완료 Read·Write·Edit·Bash 등 기본 도구 제공. MCP로 확장 가능
Layer 2 Context 직접 설계 ★ CLAUDE.md로 규칙·기억·세계관을 직접 설계. 이 장의 핵심
Layer 3 Subagent 내장 완료 Task 툴로 서브에이전트 실행. Skills가 더 가벼운 방식으로 같은 역할을 함
Layer 4 Hooks 플랫폼 관리 Claude Code처럼 스크립트를 직접 작성할 수는 없음. CLAUDE.md 규칙으로 일부 커버 가능
Layer 5 Permissions 내장 완료 Cowork가 안전한 기본값으로 관리. 사용자가 별도 설정 불필요
+Skills SKILL.md 직접 설계 ★ 전문 역할을 가진 경량 에이전트. Layer 1~3을 조합해 동작

5개 레이어 중 4개는 Cowork가 대신 처리합니다. 우리가 설계할 수 있는 것은 딱 두 가지 — CLAUDE.md와 Skills입니다. 그리고 이 두 가지만으로 Claude의 행동을 완전히 바꿀 수 있습니다.

System Prompt — 손 못 대는 하네스 기반층

Cowork에는 사용자가 볼 수도, 건드릴 수도 없는 레이어가 하나 있습니다. 바로 System Prompt입니다. 매 대화 시작 시 자동으로 로딩되는 Anthropic의 설계 기반층으로, 하네스의 가장 아래에 깔려 있습니다.

CLAUDE.md는 System Prompt 위에 얹히는 추가 컨텍스트입니다. Claude는 System Prompt의 기본 규칙을 따르면서, 동시에 CLAUDE.md에 적힌 내용도 함께 읽고 행동합니다.

CLAUDE.md · Skills · MCP — 손 댈 수 있는 세 가지

두 가지는 역할이 다릅니다. CLAUDE.md는 Claude가 항상 따르는 규칙이고, Skills는 특정 상황에서 호출되는 전문가입니다. 자동차로 비유하면 CLAUDE.md는 운전 매너이고 Skills는 목적지별 내비게이션입니다.

CLAUDE.md가 없으면 어떻게 달라지나

CLAUDE.md가 없어도 Cowork는 작동합니다. 하지만 Claude는 매 대화를 백지 상태에서 시작합니다. 말투도, 저장 위치도, 금지 사항도 — 매번 다시 말해줘야 합니다.

상황 CLAUDE.md 없을 때 CLAUDE.md 있을 때
말투·언어 매번 "한국어로 해줘" 요청 필요 자동으로 설정된 말투 유지
파일 저장 저장 위치를 매번 지정해야 함 규칙에 따라 자동 저장
작업 방식 Claude가 임의로 판단 정해진 절차대로 진행
스킬 연동 스킬 트리거 조건 없음 상황에 맞는 스킬 자동 호출
일관성 대화마다 다른 결과 어느 대화에서도 동일한 행동

CLAUDE.md는 한 번 잘 만들어두면 이후 모든 대화에서 작동합니다. 매번 긴 지시를 반복하는 대신, 그 지시를 파일 하나에 담아두는 것입니다.
