철학 — CLAUDE.md는 트리거만 남긴다

3장에서 원칙을 다뤘다면, 여기서는 실제로 이렇게 구현했습니다.

CLAUDE.md를 처음 만들 때 흔히 저지르는 실수가 있습니다. 모든 걸 한 파일 안에 다 써넣는 것입니다. 말투, 규칙, 예시, 예외 처리, 참고 자료까지 — 한 곳에 뭉쳐 있으면 수정할 때 어디를 고쳐야 할지 찾기 어렵고, 다른 프로젝트에서 재사용하기도 힘들어집니다. 역할별로 파일을 분리하고, CLAUDE.md에는 연결 고리만 남기는 방식이 이 구조의 핵심입니다.

공용폴더 — 여러 세션이 공유하는 파일들

Cowork는 세션마다 새로운 대화가 시작됩니다. 그런데 매번 같은 파일을 첨부하거나 같은 설명을 반복할 필요는 없습니다. 공용폴더 하나를 만들어두고, CLAUDE.md에 "거기 있다"고 한 번만 써두면 됩니다. 어떤 세션에서 시작해도 Claude가 그 폴더를 참조합니다.

├── mj_prompt_rules.md
├── fonts/
└── design-tokens.md

세계관.md Claude와의 관계 설정. 호칭, 말투, 기억할 것들을 담아두는 파일.
아이디어메모.md 대화 중 생각난 내용을 Claude가 즉시 기록하는 시스템. 나중에 다시 찾아볼 수 있다.
인수인계서_템플릿.md 세션이 끊길 때 다음 대화로 넘기는 인수인계서 표준 양식.
mj_prompt_rules.md 특정 작업 트리거 시 참조하는 규칙 파일. 작업별로 이런 파일을 추가할 수 있다.
fonts/ 한글 폰트 모음. SVG·이미지 작업 시 매번 챙기지 않아도 여기서 참조.
design-tokens.md 자주 쓰는 디자인 토큰 정리. 한 번 만들어두면 어떤 작업에서도 일관된 디자인 적용 가능.

fonts/, design-tokens.md처럼 매번 챙기기 번거로운 자료를 공용폴더에 두고 CLAUDE.md에 한 줄로 위치만 알려두면 됩니다. 이후로는 어떤 세션에서 시작해도 Claude가 알아서 찾아 씁니다.

Chapter 05

CLAUDE.md 구조 들여다보기

실제 사용 중인 CLAUDE.md의 섹션 구조입니다. 내용을 통째로 싣기보다, 어떤 섹션이 있고 각각이 어떤 역할을 하는지 구조만 봐도 설계 방향을 참고할 수 있습니다.

1. 기본 행동 규칙
트리거형
대화 시작 시 세계관.md를 읽는다. 답변 형식, 질문 규칙, 대화 vs 실행 구분, 메모 트리거 등 기본 행동 패턴. 일부는 트리거, 일부는 짧은 직접 규칙.
2. 제약 조건
직접 규칙형
파일 삭제 금지, 덮어쓰기 전 확인, 요청받은 것만 수정 등 Claude의 행동을 제한하는 규칙들. 여기는 파일로 빼지 않고 직접 기술.
3. 작업 워크플로우
직접 규칙형
파일 생성 순서, 작업 폴더 명명 규칙, 폴더 생성 전 기존 폴더 확인 등. 작업 단위마다 반복되는 절차를 정리해둔 섹션.
4. 스킬 & 도구 규칙
트리거형
미드저니 작업 시 mj_prompt_rules.md 참조, 한글 폰트는 fonts/ 폴더 우선 사용 등. 도구별 규칙이 생길 때마다 트리거를 추가하는 방식으로 확장.
5. 공용폴더 문서 목록
트리거형
공용폴더 안에 어떤 파일이 있는지와 각각의 역할을 한 번만 정리. Claude가 필요할 때 여기를 보고 파일 위치를 파악.
6. 인수인계서 규칙
직접 규칙형
인수인계서를 언제 작성하고, 어떻게 불러오는지에 대한 규칙. 세션 간 연결의 핵심이라 CLAUDE.md에 직접 기술.

트리거형 섹션은 "어떤 상황에서 어떤 파일을 읽어라"가 전부입니다. 직접 규칙형도 항목당 한두 줄을 넘기지 않도록 유지합니다. 분리의 목적은 효율이 아니라 관리입니다 — 고칠 것이 생겼을 때 해당 파일만 열면 된다는 것, 그게 핵심입니다.

Chapter 05

이렇게 쓰는 이유

설계 방식에는 정답이 없습니다. 하지만 이 구조로 운영하면서 확실히 좋아진 것들이 있습니다.

1
수정이 쉽다
Claude와의 관계 설정을 바꾸고 싶으면 세계관.md만 열면 됩니다. 미드저니 규칙을 업데이트하고 싶으면 mj_prompt_rules.md만 수정. CLAUDE.md를 건드릴 필요가 없습니다. 역할별로 파일이 나뉘어 있으니 수정 범위가 명확합니다.
2
어느 세션에서 시작해도 동일하게 작동한다
공용폴더가 항상 같은 위치에 있고 CLAUDE.md가 이를 참조하니까, 새 프로젝트 폴더를 만들어도 기존 세팅이 그대로 유지됩니다. fonts/, design-tokens.md도 마찬가지입니다. 한 번 설치해두면 모든 세션에서 자동으로 활용됩니다.
3
인수인계로 세션이 끊겨도 이어진다
작업이 끝날 때 인수인계서를 남기면, 다음 세션의 Claude가 그걸 읽고 그대로 이어서 진행합니다. CLAUDE.md가 인수인계서 로딩 규칙을 갖고 있기 때문에, 새 대화를 열어도 이전 맥락이 살아있습니다.

# 기본 행동 규칙 직접 규칙
## 작업 전 확인
- 요청이 모호할 때 혼자 판단하지 않는다.
선택지를 보여주고 고른 뒤 시작한다.
- 복잡한 작업은 시작 전 핵심 사항을 먼저 확인한다.
- 의견을 물으면 답하고 기다린다.
혼자 "실행 모드"로 전환하지 않는다.
모호한 요청을 혼자 해석하고 작업을 시작하는 걸 막는 규칙
## 요청한 것만 한다
- 부탁받은 것 이상을 만들지 않는다.
있으면 좋을 것 같아도 추가하지 않는다.
- 관련 없는 파일은 건드리지 않는다.
개선이 필요해 보이면 말만 하고 기다린다.
과잉 친절이 오히려 작업을 망치는 경우를 방지
## 파일 수정 규칙
- 기존 파일 덮어쓰기 전 반드시 먼저 알린다.
- 삭제는 명시적으로 "삭제해줘"라고 할 때만.
- 돌이킬 수 없는 작업은 반드시 먼저 확인한다.
데이터 안전을 위한 최소 방어선
## 답변 형식
- 불릿·번호 목록은 꼭 필요할 때만. 기본은 문장.
- 파일 생성 후 바로 열 수 있는 링크로 제공한다.
- 결과 요약은 짧게.
Cowork 특성상 파일 링크가 결과물의 핵심
