Chapter 03

직접 쓰면 왜 힘든가

CLAUDE.md를 처음 만들려는 사람 앞에는 빈 텍스트 파일 하나만 있습니다. 말투를 써야 하는지, 금지사항을 써야 하는지, 얼마나 써야 하는지 — 기준이 없으면 시작부터 막힙니다. 그리고 어떻게든 쓰더라도 세 가지 함정이 기다리고 있습니다.

1
뭘 써야 할지 모른다

말투인가, 규칙인가, 금지사항인가. 구조가 없으면 생각나는 것을 랜덤하게 적게 됩니다. 결과적으로 중요한 것은 빠지고, 없어도 되는 것은 길게 적힙니다.

2
길어질수록 효과가 떨어진다

쓰다 보면 욕심이 생겨 계속 추가하게 됩니다. 그런데 CLAUDE.md가 너무 길면 컨텍스트를 낭비하고, Claude가 중요한 규칙을 놓치기 시작합니다. 많이 쓴다고 더 잘 작동하지 않습니다.

3
한 번 쓰고 방치된다

처음에 공들여 만들어도 시간이 지나면 업데이트를 안 하게 됩니다. 작업 방식이 바뀌었는데 CLAUDE.md는 예전 규칙을 갖고 있으면, 오히려 방해가 됩니다.

문제는 글쓰기 실력이 아닙니다. 구조를 모른 채 시작하는 것입니다. 좋은 CLAUDE.md는 혼자 쓰는 문서가 아니라, Claude와 대화하며 뽑아내는 결과물입니다.

Claude에게 인터뷰받듯 만드는 방법

CLAUDE.md를 먼저 쓰려고 하지 마세요. 대신 Claude에게 이렇게 말하세요 — "내 CLAUDE.md를 만들어줘. 필요한 걸 나한테 물어봐." 그러면 Claude가 인터뷰어가 됩니다.

인터뷰 흐름 예시
Claude
어떤 언어와 말투로 답변할까요? 존댓말/반말, 이모지 사용 여부도 알려주세요.
한국어로, 친근하지만 존댓말. 이모지는 안 써도 돼.
나
Claude
파일을 만들 때 기본 저장 위치와 파일명 규칙이 있나요?
claude_work/ 폴더에, 파일명은 YYYYMMDD_내용 형식으로.
나
Claude
절대로 하면 안 되는 것, 혹은 반드시 먼저 확인해야 할 상황이 있나요?
… 몇 가지 더 묻고 나면 Claude가 답변을 정리해서 CLAUDE.md 초안을 생성합니다.

인터뷰 방식의 핵심은 구조가 Claude에게 있다는 것입니다. 나는 내 작업 방식을 말하기만 하면 됩니다. 빈 파일 앞에서 막막할 필요가 없습니다.

트리거만 남기고 나머지는 파일로 분리하라

CLAUDE.md가 완성된 뒤에도 욕심이 생깁니다. 이것도 추가하고 싶고, 저것도 쓰고 싶고. 그러다 보면 CLAUDE.md가 300줄, 500줄이 됩니다. Claude가 매 대화마다 그 내용을 전부 읽어야 하니 컨텍스트가 낭비되고, 정작 중요한 규칙은 희석됩니다.

CLAUDE.md (비대한 버전) 450줄
# 말투 규칙
한국어로 답변한다. 존댓말을 사용한다…
이모지는 쓰지 않는다. 단, 사용자가…
# 미드저니 프롬프트
--ar 16:9 기본값. 스타일은 --sref…
네거티브 프롬프트 목록: blurry…
# 파일 관리
저장 위치: claude_work/…
파일명 규칙: YYYYMMDD_…
… (계속)
CLAUDE.md (트리거만) 40줄
# 세계관 로딩
대화 시작 시 세계관.md를 읽는다.
# 미드저니 규칙
MJ 프롬프트 필요 시
mj_prompt_rules.md를 먼저 읽는다.
# 파일 관리
저장 위치·파일명 규칙은
파일관리규칙.md 참조.
권장 폴더 구조
claude_work/
├── CLAUDE.md ← 트리거만. 짧고 가볍게
└── 0.공용/
├── 세계관.md ← 말투·관계·기억
├── 아이디어메모.md ← 대화 중 떠오른 기록
├── 인수인계서_템플릿.md ← 세션 인계 표준 양식
├── mj_prompt_rules.md ← MJ 전용 규칙
├── fonts/ ← 한글 폰트 모음
└── design-tokens.md ← 디자인 토큰 참고

CLAUDE.md는 목차가 되어야 합니다. 세부 내용은 별도 파일에 두고, CLAUDE.md는 "이 상황에서 저 파일을 읽어라"는 트리거만 담습니다. 이렇게 하면 컨텍스트도 절약되고 유지보수도 쉬워집니다.

Chapter 03

다이어트 — 리뷰도 Claude한테 맡겨라

CLAUDE.md는 한 번 만들고 끝이 아닙니다. 쓰다 보면 조금씩 추가되고, 어느 순간 다시 무거워집니다. 이때도 직접 정리하려 하지 마세요. Claude에게 맡기면 됩니다.

다이어트 프롬프트 예시
기본 리뷰
"CLAUDE.md를 읽고 중복된 규칙, 더 이상 필요 없어 보이는 내용, 별도 파일로 분리하면 좋을 내용을 찾아줘."
리뷰 시 확인할 세 가지
1
중복·겹치는 규칙
같은 말이 다른 섹션에 반복되고 있지는 않은가
2
분리 가능한 세부 내용
CLAUDE.md에 있지만 별도 파일로 뺄 수 있는 상세 내용
3
현실과 맞지 않는 규칙
작업 방식이 바뀌었는데 예전 규칙이 그대로 남아 있지 않은가
CLAUDE.md에 직접 넣어두는 방법
### CLAUDE.md 최적화 규칙
- 새 섹션이 추가되거나 200줄을 넘기면,
  수정 완료 후 아래 기준으로 점검한다.
  - 중복되거나 겹치는 규칙이 있는가
  - 별도 파일로 분리할 수 있는 내용이 섞여 있는가
  - 더 이상 쓰지 않는 규칙이 있는가
- 최적화가 필요하면 이유와 방법을 제시하고
  진행 여부를 물어본다.
- 단순 문구 수정·날짜 업데이트는 검토를 생략한다.

이 규칙을 CLAUDE.md 안에 넣어두면 Claude가 스스로 다이어트 시점을 판단하고 먼저 제안합니다. 한 번 설정해두면 따로 신경 쓸 필요가 없습니다.
