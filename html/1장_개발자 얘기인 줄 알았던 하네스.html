개발자 얘기인 줄
알았던 하네스

"하네스"라는 단어를 처음 들었을 때 어떠셨나요? Claude Code, 에이전트, 병렬 실행 — 온통 개발자 냄새가 나서 "나랑은 다른 세계 얘기구나" 싶었을 겁니다. 그런데 사실 Cowork를 쓰고 있다면 이미 하네스 안에 있습니다.

Chapter 01

프롬프트로는 한계가 있다

Claude에게 "이렇게 해줘"라고 말하는 것이 프롬프트 엔지니어링입니다. 그런데 아무리 정성껏 쓴 프롬프트도 반복적인 작업, 복잡한 플로우, 세션 간 기억 유지 앞에서는 흔들립니다. 대화가 길어질수록 앞에 쓴 지시가 희미해지고, 같은 말을 계속 반복해야 하는 상황이 생깁니다.

이것이 최근 AI 엔지니어링의 무게중심이 빠르게 하네스 엔지니어링으로 이동한 이유입니다. 에이전트 연구의 핵심 키워드로 자리잡았습니다.

하네스란 무엇인가

"하네스(Harness)"는 원래 말을 제어하는 마구(馬具)나 등산가의 안전벨트를 뜻합니다. 소프트웨어에서는 무언가를 실제로 작동하게 만드는 외부 골격, 즉 테스트 하네스처럼 쓰였습니다. AI 세계에서 하네스는 LLM을 감싸는 모든 레이어를 가리킵니다.

TOOLS
파일 읽기·쓰기, 명령 실행, API 호출
CONTEXT
무엇을 기억하고, 잊고, 압축할 것인가
CONTROL LOOP
언제 호출하고, 멈추고, 재시도할 것인가
PERMISSIONS
허용·차단·승인이 필요한 작업의 경계
MEMORY
세션을 넘어 유지되는 지식과 규칙

프롬프트는 이 다섯 가지 중 하나의 입력일 뿐입니다. 하네스가 약하면 아무리 정성껏 쓴 프롬프트도 성능의 천장에 부딪힙니다.

OODA 루프로 보는 하네스의 역할

LLM은 혼자서는 다음 토큰을 생성하는 것밖에 할 수 없습니다. 실제 작업을 해결하려면 군사 전략에서 빌려온 OODA 루프가 필요합니다. 주목할 점은 4단계 중 3단계가 하네스의 담당이라는 것입니다.

OBSERVE
👁
관찰
파일·DB·환경 읽기
하네스
ORIENT
🧭
방향 설정
정보 가공·LLM 전달
하네스
DECIDE
🧠
결정
다음 행동 선택
LLM
ACT
⚡
실행
명령·API 실행
하네스
←    루프는 계속 반복됩니다    →

4단계 중 3단계 — Observe, Orient, Act — 가 하네스의 몫입니다. LLM이 담당하는 것은 Decide 하나뿐입니다. 하네스의 품질이 곧 에이전트의 품질입니다.

이 구조가 보이면 이후 Cowork에서 CLAUDE.md와 Skills가 어느 단계를 담당하는지 훨씬 명확하게 이해됩니다.

단계 담당 Cowork에서는
Observe 하네스 작업 폴더·업로드 파일 읽기, MCP 연결 데이터 수집
Orient CLAUDE.md 규칙·말투·세계관·스킬 트리거를 컨텍스트로 전달
Decide Claude (LLM) 어떤 스킬을 쓸지, 어떻게 답할지 판단
Act 하네스 파일 생성·수정, 스킬 실행, 외부 도구 호출

Chapter 01

Claude Code의 하네스: 5개 레이어

Claude Code는 Anthropic이 직접 설계한 대표적인 에이전트 하네스입니다. 내부 구조를 5개의 핵심 레이어로 정리해보면 다음과 같습니다.

레이어 구성 요소 역할
Layer 1 Tools Read, Edit, Bash, Grep… 실제 도구 — 파일 조작, 명령 실행
Layer 2 Context CLAUDE.md · Memory 프로젝트 규칙 · 기억 · 세계관 ← 핵심
Layer 3 Subagent Agent 툴 · Skills 별도 컨텍스트로 작업 위임. Cowork에서는 Skills가 이 자리를 경량으로 활용
Layer 4 Hooks settings.json 스크립트 자동 가드레일 — 특정 행동 전·후 자동 실행
Layer 5 Permissions allow · deny · ask 허용·차단·승인 필요한 작업의 경계 설정

흔한 오해: 멀티 에이전트 = 하네스?

최근 멀티 에이전트 병렬 실행 콘텐츠가 쏟아지면서 "하네스 = 멀티 에이전트"라는 오해가 생겼습니다. 하지만 표에서 보듯 멀티 에이전트(Subagent)는 Layer 3 하나일 뿐입니다.

❌ 오해
하네스 = 여러 AI가 동시에 병렬로 일하는 구조
✓ 실제
하네스 = 모델을 감싸는 모든 환경 설계
멀티 에이전트는 그 일부

다음 장에서는 Cowork에서 이 5레이어 중 어느 부분을 우리가 직접 설계할 수 있는지 살펴봅니다. 스포일러: 핵심은 Layer 2 — CLAUDE.md입니다.
